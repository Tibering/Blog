# 知识库

## 第 7 章 `FD` 公理化

> _[那些] 真实而坚实的活的公理_
>
> — `Francis Bacon`：《`The New Organon`》（1620）

我已经多次`提及`这样一个观点：某些 `FD` 蕴含其他 `FD`；现在是时候更具体地讨论了。不过首先，我需要引入一些记号——这些记号（a）减少了形式证明等场景中所需的击键次数，并且（b）有时也能帮助我们既见树木又见森林。

如您可能记得，我在第 `5` 章中陈述的 `Heath 定理` 包含以下句子：_设 `XY` 表示 `X` 与 `Y` 的并集，`XZ` 同理。_ 我想要引入的记号基本上只是这个简单想法的扩展（它略显不合逻辑，但非常方便）。具体来说，该记号使用形如 _`XY`_ 的表达式来表示：

- 如果 _`X`_ 和 _`Y`_ 是单个属性名称，则表示 `{`_`X`_`}` 与 `{`_`Y`_`}` 的并集
- 如果 _`X`_ 和 _`Y`_ 是属性名称的集合，则表示 _`X`_ 与 _`Y`_ 的并集
- 如果 _`X`_ 是单个属性名称而 _`Y`_ 表示此类名称的集合（或者 _`X`_ 和 _`Y`_ 的角色互换），则表示 `{`_`X`_`}` 与 _`Y`_ 的并集（或 _`X`_ 与 `{`_`Y`_`}` 的并集）

它还允许在 `FD` 等场景中，如果 _`X`_ 表示单个属性，则将 `{`_`X`_`}` 简写为 _`X`_。_注意_：为方便起见，从现在起我将此记号称为 _`Heath 记号`_（但我必须明确指出，`Heath` 本人在其论文中实际上并未使用它）。

### `Armstrong 公理`

我们已经看到，形式上讲，`FD` 只是一个表达式：具体来说，是形如 _`X`_ → _`Y`_ 的表达式，其中 _`X`_ 和 _`Y`_ 是集合（实际上是属性名称的集合，但从形式观点来看，集合由什么组成其实并不重要）。现在，假设我们给定某个 `FD` 集合（设为 _`F`_）。那么我们可以应用某些形式化的`推理规则` 从 _`F`_ 中的 `FD` 推导出更多的 `FD`——这些 `FD` 被 _`F`_ 中的 `FD` 所`蕴含`，意味着如果 _`F`_ 中的 `FD` 在某个给定的 `关系变量` 中成立，那么推导出的 `FD` 也在该 `关系变量` 中成立。这些规则最初由 `Armstrong` 于 1974 年提出，因此通常被称为 _`Armstrong 推理规则`_ 或（更常见地）称为 _`Armstrong 公理`_。它们可以用多种等价方式陈述，以下可能是最简单的一种：

1.  _`自反律`_：如果 _`Y`_ 是 _`X`_ 的子集（即 _`Y`_ ⊆ _`X`_），则 _`X`_ → _`Y`_。
2.  _`增广律`_：如果 _`X`_ → _`Y`_，则 _`XZ`_ → _`YZ`_。
3.  _`传递律`_：如果 _`X`_ → _`Y`_ 且 _`Y`_ → _`Z`_，则 _`X`_ → _`Z`_。

请注意，鉴于 `FD` 的预期解释，这些规则在直观上是合理的。也就是说，由于我们知道 `FD`"意味着什么"，我们可以很容易地看出，例如，如果 `FD` _`X`_ → _`Y`_ 和 _`Y`_ → _`Z`_ 都在 `关系变量` _`R`_ 中成立，那么 `FD` _`X`_ → _`Z`_ 也必须成立。（`供应商关系变量` `S` 说明了这条特定规则——`FD` `{SNO}` → `{CITY}` 和 `{CITY}` → `{STATUS}` 都在该 `关系变量` 中成立，因此 `FD` `{SNO}` → `{STATUS}` 也成立。）

因此这些规则是合理的。但更重要的是，它们既是`可靠的` 又是`完备的`。`可靠性` 和 `完备性` 是在一般形式系统中经常遇到的概念。在本文考虑的特定形式系统中，它们的含义如下：

- _`完备性`_：如果 `FD` _`f`_ 被给定集合 _`F`_ 中的 `FD` 所蕴含，那么它可以通过这些规则从 _`F`_ 中的 `FD` 推导出来。（重复一遍，说某个 `FD` _`f`_ 被某个集合 _`F`_ 中的 `FD` 所蕴含，意思是如果 _`F`_ 中的 `FD` 成立，那么 _`f`_ 也成立。）
- _`可靠性`_：如果 `FD` _`f`_ 不被给定集合 _`F`_ 中的 `FD` 所蕴含，那么它无法通过这些规则从 _`F`_ 中的 `FD` 推导出来。<sup>1</sup>

因此，这些规则构成了所谓的 `FD` 的`公理化`。因此，它们可以用来推导任何给定 `FD` 集合 _`F`_ 的所谓`闭包` _`F`_<sup>+</sup>。以下是定义：

> **`定义`（`FD 集合的闭包`）**：设 _`F`_ 是一个 `FD` 集合。那么 _`F`_ 的闭包 _`F`_<sup>+</sup> 是由 _`F`_ 中的 `FD` 所蕴含的所有 `FD` 的集合。

此外，推导过程可以`机械化`；也就是说，`Armstrong 规则` 可以被纳入（例如）设计工具中，该工具给定在某个 `关系变量` _`R`_ 中成立的 `FD` 集合 _`F`_，将能够计算该集合 _`F`_ 的闭包 _`F`_<sup>+</sup>，换句话说，即在该 `关系变量` 中成立的所有 `FD` 的完整集合。这种情况的重要性应该是显而易见的。

### 附加规则

可以从原始的三条规则推导出若干附加的推理规则，其中包括以下规则。这些附加规则可用于简化从 _`F`_ 计算 _`F`_<sup>+</sup> 的实际任务。以下是一些示例：

4.  _`自决定律`_：_`X`_ → _`X`_。
5.  _`合并律`_：如果 _`X`_ → _`Y`_ 且 _`X`_ → _`Z`_，则 _`X`_ → _`YZ`_。
6.  _`合成律`_：如果 _`X`_ → _`Y`_ 且 _`Z`_ → _`W`_，则 _`XZ`_ → _`YW`_。
7.  _`分解律`_：<sup>2</sup> 如果 _`X`_ → _`YZ`_，则 _`X`_ → _`Y`_ 且 _`X`_ → _`Z`_。

在下一节中，我将展示如何从原始的三条规则推导出这四条规则。不过首先，让我举几个例子来说明这些规则（原始规则和附加规则）在实践中如何使用。作为第一个例子，假设我们给定一个具有属性 _`A`_、_`B`_、_`C`_、_`D`_、_`E`_、_`F`_ 的 `关系变量` _`R`_，并且我们被告知以下 `FD` 在该 `关系变量` 中成立：

```
A  → BC
B  → E
CD → EF
```

我现在将展示 `FD` _`AD`_ → _`F`_ 也在 _`R`_ 中成立（我认为您会同意，这个事实并非一目了然）。<sup>3</sup> 以下是证明：

1.  _`A`_ → _`BC`_（给定）
2.  _`A`_ → _`C`_（1，分解律）
3.  _`AD`_ → _`CD`_（2，增广律）
4.  _`CD`_ → _`EF`_（给定）
5.  _`AD`_ → _`EF`_（3 和 4，传递律）
6.  _`AD`_ → _`F`_（5，分解律）

作为第二个例子，回顾第 `6` 章中 `不可约覆盖` 的概念。提醒您一下，（a）给定 `FD` 集合 _`F`_ 的一个`覆盖` 是一个 `FD` 集合 _`C`_，使得 _`F`_ 中的每个 `FD` 都被 _`C`_ 中的 `FD` 所蕴含，并且（b）该覆盖 _`C`_ 是`不可约的` 当且仅当它具有以下所有属性：

1.  _`单一依赖项`_：_`C`_ 中的每个 `FD` 在右侧只有一个属性。
2.  _`不可约决定因素`_：不能从 _`C`_ 中任何 `FD` 的左侧丢弃任何属性而不丢失 _`C`_ 是 _`F`_ 的覆盖 这一属性。
3.  _`无冗余 FD`_：不能从 _`C`_ 中丢弃任何 `FD` 而不丢失 _`C`_ 是 _`F`_ 的覆盖 这一属性。

现在，我在前一章中（隐含地）假设每个 `FD` 集合 _`F`_ 都有一个不可约覆盖。事实上，这很容易理解：

- 得益于分解律，我们可以不失一般性地假设 _`F`_ 中的每个 `FD` 都有一个单一右侧。
- 接下来，对于 _`F`_ 中的每个 `FD`，检查左侧的每个属性 _`A`_；如果从该左侧删除 _`A`_ 对闭包 _`F`_<sup>+</sup> 没有影响，则从该左侧删除 _`A`_。
- 对于 _`F`_ 中剩余的每个 `FD`，如果从 _`F`_ 中删除该 `FD` 对闭包 _`F`_<sup>+</sup> 没有影响，则从 _`F`_ 中删除该 `FD`。

_`F`_ 的最终版本是不可约的，并且是原始版本的覆盖。

以下是一个具体示例，用于说明实际查找不可约覆盖的过程。设给定的 `FD` 集合（所有这些 `FD` 假定在某个具有属性 _`A`_、_`B`_、_`C`_、_`D`_ 的 `关系变量` _`R`_ 中成立）如下：

```
A  → BC
B  → C
A  → B
AB → C
AC → D
```

那么以下过程将为该给定集合生成一个不可约覆盖：

1.  首先，重写 `FD` 使得每个 `FD` 都有一个单一右侧：

    ```
    A  → B
    A  → C
    B  → C
    A  → B
    AB → C
    AC → D
    ```

    现在注意到 `FD` _`A`_ → _`B`_ 出现了两次，因此可以删除其中一个。

2.  属性 _`C`_ 可以从 `FD` _`AC`_ → _`D`_ 的左侧删除，因为我们有 _`A`_ → _`C`_，所以通过增广律有 _`A`_ → _`AC`_，并且我们给定 _`AC`_ → _`D`_，所以通过传递律有 _`A`_ → _`D`_；因此 _`AC`_ → _`D`_ 左侧的 _`C`_ 是冗余的。

3.  `FD` _`AB`_ → _`C`_ 可以删除，因为同样我们有 _`A`_ → _`C`_，所以通过增广律有 _`AB`_ → _`CB`_，所以通过分解律有 _`AB`_ → _`C`_。

4.  `FD` _`A`_ → _`C`_ 被 `FD` _`A`_ → _`B`_ 和 _`B`_ → _`C`_ 共同蕴含，因此可以删除。

我们剩下：

```
A → B
B → C
A → D
```

这个集合是不可约的。

### 证明附加规则

如前所述，在本节中我将展示如何从原始的规则 1-3 推导出规则 4-7。

4.  _`自决定律`_：_`X`_ → _`X`_。
    **`证明`**：由自反律直接得出。

5.  _`合并律`_：如果 _`X`_ → _`Y`_ 且 _`X`_ → _`Z`_，则 _`X`_ → _`YZ`_。
    **`证明`**：_`X`_ → _`Y`_（给定），因此通过增广律有 _`X`_ → _`XY`_；同样 _`X`_ → _`Z`_（给定），因此通过增广律有 _`XY`_ → _`YZ`_；因此通过传递律有 _`X`_ → _`YZ`_。

6.  _`合成律`_：如果 _`X`_ → _`Y`_ 且 _`Z`_ → _`W`_，则 _`XZ`_ → _`YW`_。
    **`证明`**：_`X`_ → _`Y`_（给定），因此通过增广律有 _`XZ`_ → _`YZ`_；同样，_`Z`_ → _`W`_（给定），因此通过增广律有 _`YZ`_ → _`YW`_；因此通过传递律有 _`XZ`_ → _`YW`_。

7.  _`分解律`_：如果 _`X`_ → _`YZ`_，则 _`X`_ → _`Y`_ 且 _`X`_ → _`Z`_。
    **`证明`**：_`X`_ → _`YZ`_（给定）且通过自反律有 _`YZ`_ → _`Y`_；因此通过传递律有 _`X`_ → _`Y`_（同理可得 _`X`_ → _`Z`_）。

### 另一种闭包

回顾一下，`FD` 集合 _`F`_ 的闭包 _`F`_<sup>+</sup> 是由 _`F`_ 中的 `FD` 所蕴含的所有 `FD` 的集合。现在，原则上我们可以通过重复应用 `Armstrong 规则`（和/或从中推导出的规则）直到它们不再生成新的 `FD` 来从 _`F`_ 计算 _`F`_<sup>+</sup>。然而在实践中，几乎没有必要计算 _`F`_<sup>+</sup> 本身（这也许也好，因为刚才概述的过程效率确实不高）。但现在我想展示如何计算 _`F`_<sup>+</sup> 的某个重要子集：即由给定`决定因素` 组成的所有 `FD` 构成的子集。更准确地说，我将展示给定一个`标题` _`H`_、_`H`_ 的一个子集 _`Z`_，以及关于 _`H`_ 的 `FD` 集合 _`F`_，我们如何计算所谓的 _`Z`_ 在 _`F`_ 下的`闭包` _`Z`_<sup>+</sup>。以下是定义：

> **`定义`（`属性集合的闭包`）**：设 _`H`_ 是一个`标题`，设 _`Z`_ 是 _`H`_ 的一个子集，设 _`F`_ 是关于 _`H`_ 的 `FD` 集合。那么 _`Z`_ 在 _`F`_ 下的闭包 _`Z`_<sup>+</sup> 是 _`H`_ 的最大子集 _`C`_，使得 _`Z`_ → _`C`_ 被 _`F`_ 中的 `FD` 所蕴含。

顺便提一下，请注意我们现在有两种闭包（尽量不要混淆它们！）：`FD 集合的闭包`，以及`在 FD 集合下的属性集合的闭包`。<sup>4</sup> 还要注意，我们对两者使用相同的"上标加号"记号。

以下是计算 _`Z`_ 在 _`F`_ 下的闭包 _`Z`_<sup>+</sup> 的一个简单`伪代码` 算法：

```
Z⁺ := Z ;
do "forever" ;
    for each FD X → Y in F
        do ;
            if X is a subset of Z⁺
            then replace Z⁺ by the union of Z⁺ and Y ;
        end ;
    if Z⁺ did not change on this iteration
    then quit ;  /* computation complete */
end ;
```

让我们做一个例子。假设给定的`标题` 是 _`ABCDEG`_，我们想要计算属性集合 _`AB`_ 在以下 `FD` 集合 _`F`_ 下的闭包 _`AB`_<sup>+</sup>：

```
A  → BC
E  → CG
B  → E
CD → EG
```

现在让我们逐步执行该算法：

1.  首先，我们将结果 _`AB`_<sup>+</sup> 初始化为属性集合 _`AB`_。
2.  我们现在遍历内层循环四次，每次对应一个给定的 `FD`。在第一次迭代（针对 `FD` _`A`_ → _`BC`_）时，我们发现`决定因素` _`A`_ 确实是迄今为止计算的 _`AB`_<sup>+</sup> 的子集，因此我们将属性（_`B`_ 和）_`C`_ 添加到结果中。_`AB`_<sup>+</sup> 现在是集合 _`ABC`_。
3.  在第二次迭代（针对 `FD` _`E`_ → _`CG`_）时，我们发现`决定因素` _`E`_`不是` 迄今为止计算的结果的子集，因此结果保持不变。
4.  在第三次迭代（针对 `FD` _`B`_ → _`E`_）时，我们将 _`E`_ 添加到 _`AB`_<sup>+</sup>，因此它现在具有值 _`ABCE`_。
5.  在第四次迭代（针对 `FD` _`CD`_ → _`EG`_）时，_`AB`_<sup>+</sup> 保持不变。
6.  现在我们再次遍历内层循环四次。在第一次迭代时，结果保持不变；在第二次迭代时，它扩展到 _`ABCEG`_；在第三和第四次迭代时，它保持不变。
7.  现在我们再次遍历内层循环四次。结果保持不变，因此整个过程终止，最终 _`AB`_<sup>+</sup> = _`ABCEG`_。

好吧，我希望您能从这个例子中看出，给定 _`H`_、_`F`_ 和 _`Z`_，计算 _`Z`_<sup>+</sup> 本质上是直截了当的。重要的是：给定某个 `FD` 集合 _`F`_（关于某个`标题` _`H`_），我们可以很容易地判断某个特定 `FD` _`X`_ → _`Y`_（关于同一个`标题` _`H`_）是否被 _`F`_ 所蕴含，因为当且仅当 _`Y`_ 是 _`X`_ 在 _`F`_ 下的闭包 _`X`_<sup>+</sup> 的子集时，它才会被蕴含。换句话说，我们现在有一种简单的方法来确定给定的 `FD` _`X`_ → _`Y`_ 是否在 _`F`_ 的闭包 _`F`_<sup>+</sup> 中，而无需实际计算 _`F`_<sup>+</sup>。

从`属性集合的闭包` 的定义也可以得出，`关系变量` _`R`_ 的`超键` 正是那些`标题` 的子集 _`SK`_，使得 _`SK`_ 在相关 `FD` 集合下的闭包 _`SK`_<sup>+</sup> 是 _`R`_ 的整个`标题`。

### 答案

7.1 参见本章正文。

7.2 `FD` 集合 _`F`_ 的闭包 _`F`_<sup>+</sup> 是由 _`F`_ 中的 `FD` 所蕴含的所有 `FD` 的集合。仅包含 `FD` `{SNO,PNO}` → `{QTY}` 的 `FD` 集合的闭包在第 `4` 章练习 4.1 的答案中给出。

7.3 `FD` _`X`_ → _`Y`_ 被满足当且仅当每当两个`元组` 在 _`X`_ 上一致时，它们在 _`Y`_ 上也一致（我故意以相当松散的形式给出这个定义）。因此：

- 如果两个`元组` 在 _`X`_ 上一致，它们肯定在 _`X`_ 的每个子集 _`Y`_ 上一致，因此自反律是合理的。
- 如果两个`元组` 在 _`XZ`_ 上一致，它们肯定在 _`Z`_ 上一致。它们也肯定在 _`X`_ 上一致，因此如果 _`X`_ → _`Y`_ 被满足，则在 _`Y`_ 上也一致；因此它们在 _`YZ`_ 上一致，所以增广律是合理的。
- 如果两个`元组` 在 _`X`_ 上一致且 _`X`_ → _`Y`_ 被满足，则它们在 _`Y`_ 上一致。如果它们在 _`Y`_ 上一致且 _`Y`_ → _`Z`_ 被满足，则它们在 _`Z`_ 上一致。因此传递律是合理的。

7.5 设 _`U`_ 表示 _`Z`_ 和 _`Y`_ 的交集，设 _`V`_ 表示 _`Z`_ 与 _`Y`_ 的差集 _`Z – Y`_（按此顺序）。那么：

1.  _`X`_ → _`Y`_（给定）
2.  _`Z`_ → _`W`_（给定）
3.  _`X`_ → _`U`_（1，分解律）
4.  _`XV`_ → _`UV`_（3，增广律）
5.  _`XV`_ → _`Z`_（简化 4）
6.  _`XV`_ → _`W`_（5, 2，传递律）
7.  _`XV`_ → _`YW`_（1, 6，合成律；证明完成）

证明中使用的规则已在注释中指出。以下规则都是 `Darwen 定理` 的特例：合并律、传递律和增广律。以下有用规则也是如此：

如果 _`X`_ → _`Y`_ 且 _`XY`_ → _`Z`_，则 _`X`_ → _`Z`_。

_注意_：后者有时被称为 _`伪传递律`_ 的特例，其一般形式如下：

_`伪传递律`_：如果 _`X`_ → _`Y`_ 且 _`YW`_ → _`Z`_，则 _`XW`_ → _`Z`_。

1.  _`AB`_ → _`C`_
2.  _`C`_ → _`A`_
