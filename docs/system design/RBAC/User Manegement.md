### 模块模型

@startuml
entity User {
+Long userId : 用户ID
+String username : 用户名
+String nickname : 昵称
+String email : 邮箱
+String phone : 手机号
+Integer status : 状态
}

note right of User
状态说明:
1: 启用
0: 禁用
end note
@enduml

### 接口设计

- 新增用户
- 查找用户
- 启用/禁用用户
- 编辑用户
- 分页查询用户[进入页面时获取 | 刷新列表获取]
- 分配角色

### 组件

按钮：新增用户 | 启用/禁用 | 编辑 | 分配角色
输入框：查询用户（支持用户名/昵称/手机号模糊搜索）
表单：新增用户 | 编辑用户 | 分配角色（多选下拉框）

### 功能

**新增用户**：

1. 填写用户基本信息（用户名、昵称、邮箱、手机号）。
2. 设置初始密码（或生成随机密码）。
3. **分配角色**：远程加载角色列表，勾选初始角色。
4. 提交保存：同时写入 `sys_user` 表和 `sys_user_role` 关联表（事务控制）。

**查找用户**：

- 支持根据 `userId` 精确查找。
- 支持根据 `username`、`nickname` 或 `phone` 进行模糊匹配查找。

**启用/禁用用户**：

- 切换用户的 `status` 字段（1: 启用, 0: 禁用）。
- 禁用后，该用户将无法登录系统，且现有会话失效。

**编辑用户**：

- 更新用户的基本信息（昵称、邮箱、手机号等）。
- _注意：通常不允许修改 `username`，若需修改需确保唯一性校验。_
- 可同时调整用户关联的角色信息（见“分配角色”）。

**分配角色**：

1. 打开角色分配弹窗。
2. 远程加载所有可用的角色列表。
3. 勾选当前用户已拥有的角色。
4. 提交更新 `sys_user_role` 关联表（先删后增，事务控制）。

**分页查询用户**：

- 进入页面或点击刷新时，携带分页参数（pageNum, pageSize）及搜索条件请求后端。
- 返回用户列表（通常包含角色名称汇总字段以便展示）及总数。
