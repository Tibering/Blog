现在使用AI开发越来越多，但是想要快速写出一个东西很难。本次踩坑写这些东西。不考虑UI/UX。我就是一个臭写后端的。

到最后肯定要回归到传统软件设计，并不存在使用AI一个软件跑到底。除非你AI跑一遍就不会再看了，因为后期软件要迭代。

当然，还有一个方式，数据模型简单直接可以通过AI迭代。毕竟功能很简单。

### 工具准备

GIT && DOcker: git用户git瞎几把乱搞时候把文件回滚到之前内容，docker可以直接让SQL快速写入数据库。

代码生成器：Mybatis Generator && Mybatis plus代码生成 && IDE的代码生成插件 && Restful API代码生成（openapi gen）

Viber Coding工具：各种代码生成器。

### 步骤

首先快速把应用的数据模型跑出来，并且用docker跑起来，如果用AI生成，最好使用相关提示词，在生成时注意收集相关提示词。

快速建立前后端。mapper层可以用代码生成器跑出来，然后controller跑出来。最后跑出页面。

设计好页面收集相关接口信息设计好模块，并且修改页面，修改接口。

现在你有了原型图。接口前后端分离的接口信息。就可以开始选择后端的系统架构，前端的框架。

复制原型图总结接口，然后不断问AI调整接口信息，最后做好restFul API。
